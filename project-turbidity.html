River Turbidity Assessment

Used satellite imagery to map turbidity patterns over time and identify potential siltation hotspots. This case study is independent and uses only public-domain, synthetic, or sanitized examples.

Highlights

What this assessment was designed to answer and how it’s structured.

Objective

  • Map spatial turbidity patterns and detect hotspots.
  • Track changes through time (seasonality / anomalies).
  • Produce clear maps and summaries for interpretation.

Approach

  • Assemble imagery collection and apply QA masking.
  • Compute turbidity proxy (e.g., NDTI) and aggregate.
  • Summarize trends and create monthly/seasonal maps.

Tools

  • Google Earth Engine for processing at scale.
  • QGIS/ArcGIS for cartography (as needed).
  • Python for post-processing and plotting (optional).

My role

  • Designed the workflow and QC approach.
  • Implemented index extraction + map outputs.
  • Produced interpretation-ready visuals.

Key takeaways

  • Monthly composites reduce noise and improve interpretability.
  • Clear QA/QC steps are critical for defensible results.
  • Maps + trend summaries work well for non-technical audiences.
Module 1

Data assembly & QA

Build the image collection and apply masking to improve signal quality.

  • Select imagery and time range (e.g., multi-year).
  • Cloud/shadow masking and water masking as appropriate.
  • Standardize bands and resolution handling.
Module 2

Index computation

Compute turbidity proxy (e.g., NDTI) and generate composites.

  • Calculate index per scene and harmonize outputs.
  • Create monthly/seasonal composites.
  • Extract statistics for AOIs / segments.
Module 3

Outputs & interpretation

Deliver maps and summaries that support interpretation and reporting.

  • Hotspot maps + change detection summaries.
  • Time-series plots (monthly/seasonal).
  • Export maps and tables for reporting packs.
