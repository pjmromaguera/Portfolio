Ocean Eddy Detection and Lifecycle Tracking

Processed multi-year NetCDF datasets to detect mesoscale eddies, extract physical parameters, and track lifecycle evolution across time. This case study uses only public-domain, synthetic, or sanitized examples.

Highlights

What the pipeline produces and how it’s organized end-to-end.

Objective

  • Detect mesoscale eddies from gridded ocean data.
  • Extract physical metrics and track eddy lifecycles.
  • Summarize statistics and visualize trajectories.

Approach

  • Ingest NetCDF data and standardize coordinates/time.
  • Apply detection criteria and label eddy features.
  • Track features through time and compute metrics.

Tools

  • Python: xarray, NumPy, pandas for processing.
  • Geospatial plotting for maps/trajectories.
  • Workflow automation for batch runs.

My role

  • Built the full pipeline (ingest → detect → track).
  • Defined outputs and diagnostics for QC.
  • Created summary visuals for interpretation.

Key takeaways

  • Consistent preprocessing is essential for stable tracking.
  • Diagnostics help validate detection sensitivity.
  • Trajectory + metric summaries communicate behavior clearly.
Module 1

Data preparation

Standardize time, grids, and masks to ensure consistent detection.

  • Load multi-year NetCDF files and harmonize coordinates.
  • Subset region/time and apply QC rules.
  • Prepare variables needed for detection.
Module 2

Detection

Identify candidate eddy structures and label features per timestep.

  • Apply detection criteria (thresholds/shape constraints).
  • Label and store per-feature attributes.
  • Generate diagnostics to verify sensitivity.
Module 3

Tracking & outputs

Connect features through time and summarize lifecycles.

  • Associate features across timesteps to build trajectories.
  • Compute lifecycle metrics (duration, intensity, displacement).
  • Export maps/plots and summary tables.
